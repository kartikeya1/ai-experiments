The premise

Same mission, two attempts at getting it right.

Both apps solve the exact same problem: give a professional trip designer a fast, offline canvas to structure a client itinerary — Trip → Sections → Items → Alternatives — and export a polished presentation. What changed between them is a study in editing your own product down to its essence.

Version 01 · Aurora Voyage

Aurora Voyage

“Premium Itinerary Studio”

The first realization of the idea. Ambitious and broad — a classic three-file web app (HTML + CSS + JS) with a light, studio-style workspace, multi-trip management, and a real Leaflet map.

  • Multi-file architecture: index.html · styles.css · app.js
  • Light-first “studio” UI, av- component system
  • Multiple trips with a switcher dropdown
  • Leaflet + OpenStreetMap interactive map
  • Board & Timeline views, undo/redo, AI-compat mode
Version 02 · Voyagr

Voyagr

“Premium Travel Itinerary Design”

The rebuild. Everything learned from v1, distilled into a single portable HTML file with a confident dark, editorial identity — and new capabilities that make it genuinely useful for real client work.

  • One self-contained file — open or deploy anywhere
  • Branded dark aesthetic: Cormorant serif + gold accent
  • Focused single-trip depth over multi-trip breadth
  • New: Packing list, Trip Stats, conflict detection
  • Truly offline map (no external dependency)

The thinking

Five decisions that defined v2.

Each move traded something from v1 for something better — the kind of editing that only becomes obvious after you've shipped the first version.

📦

Consolidate

3 files → 1 file

Collapsing HTML, CSS and JS into one document made the app trivially portable — email it, host it, run it from disk. Distribution became a non-issue.

🎨

Commit to a look

generic light → editorial dark

v1 looked like a tool. v2 looks like a product — a deliberate dark palette, a serif display face, and a single gold accent that says “premium.”

🎯

Narrow the focus

many trips → one, deeply

Multi-trip management added surface area without depth. v2 does one trip at a time — and spends the saved complexity on packing, stats and conflicts.

🔌

Earn “offline”

Leaflet CDN → no dependency

The v1 map needed the network. v2 replaces it with an ordered location listing that works truly offline — honest to the “offline-first” promise.

✨

Add what clients need

notes → real tools

Packing checklists, a live stats dashboard, and time-conflict warnings turned a pretty canvas into something a professional can actually hand to a client.

How it unfolded

The arc from v1 to v2.

Version 01

Aurora Voyage ships

The idea proven: a premium, offline itinerary studio with the full Trip → Section → Item → Alternative model, drag-and-drop, a real map, and multi-trip support across three files.

The feedback loop

Rounds of input

Using it for real surfaced the truth: the breadth (multi-trip, heavy map) mattered less than depth, portability, and a look that felt like a finished product. The direction became clear.

Version 02

Voyagr — the rebuild

Rebuilt as a single file with a confident editorial identity, a focused single-trip model, and the tools that actually close a client deal: packing, stats, conflict detection, and a cleaner export.
